Ex 01: Moteur de template.

A quoi sert le moteur de template

Le moteur de template permet de rendre dynamique les pages html en y intégrant diverses des bouts de codes qui seront processés dans un premier temps et remplacés par le html correspondant. Il existe ainsi plusieurs fonctionnalités répandues qui sont listées ci-dessous.

Les blocs: {% block block_title %}{% endblock block_title %}

Afin d'éviter la répétition de code (principe DRY) entre plusieurs pages html, il est possible d'intégrer le code d'un template dans un autre selon un principe similaire au principe d'héritage. Le template "mère" va contenir tout le code commun aux templates "filles". Dans ce templates, certains blocs sont définis. Ils seront remplis différemment selon les templates "filles".
Les blocs sont définis par ces "balises": {% block block_title %}{% endblock block_title %}. Il n'y a rien entre ces balises dans le template "mère".
Afin de lier deux templates selon le principe mère-fille, le template fille doit appeler/étendre/hériter du template "mère" avec la balise {% extends 'template_mere_path' %}. Le moteur de templates remplacera ces balises par tout le code compris entre les balises du même nom du template "fille".

Les boucles for ... in: {% for x in list}{% endfor %}

Le moteur de template interprete comme dans python les boucles. Le contenu entre les balises sera interprété pour chaque itération de la boucle. Cela est très utile notamment pour les listes et les tableaux dont les variables dependent de la list.

Les structures de contrôle if: {% if condition %}{% endif %}

La structure de contrôle permet d'afficher un contenu uniquement si la condition est remplie. La balise {% else %} permet également de faire un choix entre 2 types de contenu selon cette condition.

L'affichage de variables passées par contexte: {{ variable }}

Très souvent, dans du code html on ne veut pas mettre directement en dur un contenu. On peut utiliser une variable qui est passée par contexte dans la view. Cette variable a pu être traitée en python auparavant en fonction de la requête de l'utilisateur par exemple. On peut ajouter un filtre à la variable comme ceci : {{ variable | filtre }}. Il existe different type de filtres qui modifient les variables (majuscule, format de dates...).
